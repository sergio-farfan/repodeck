# RepoDeck v1.1 — Design Spec

## Context

RepoDeck v1 shipped and is published (private) at github.com/sergio-farfan/repodeck. On first launch Sergio hit five gaps: the app shows the generic placeholder icon; there's no appearance/theme control; the Changes and History sections have a fixed split; there's no way to search history; and no Settings window. This spec covers the v1.1 feature set that closes those gaps.

Decisions confirmed with the user:
- **Icon concept:** "Fanned status cards" on a green→emerald squircle.
- **Customization scope:** curated — appearance mode, accent color, UI font family, monospace font family, base font size (no per-element color editor).
- **History search axes:** all four — commit message, author, file path, content (pickaxe).

## Feature 1 — App icon

Reproducible, dependency-free pipeline:
- `Scripts/make-icon.swift` — a Swift/CoreGraphics program (run via `swift Scripts/make-icon.swift <out.png>`) that draws the icon at 1024×1024: a green→emerald diagonal-gradient rounded-square (macOS squircle/superellipse), three white rounded-rect cards fanned with slight rotation/offset and soft shadows, the top card bearing a small accent status dot. Art must stay legible at 32px (keep detail minimal — cards + one dot, no fine checkmark).
- `Scripts/make-iconset.sh` — renders the master PNG, then `sips -z` to the standard iconset sizes (16,32,128,256,512 @1x/@2x) into `AppIcon.iconset/`, then `iconutil -c icns` → `Sources/RepoDeck/Resources/AppIcon.icns`. Commit the generated `.icns`.
- Wiring:
  - `Package.swift`: add `resources: [.copy("Resources/AppIcon.icns")]` to the `RepoDeck` executable target.
  - `AppDelegate.applicationDidFinishLaunching`: `NSApp.applicationIconImage = NSImage(contentsOf: Bundle.module.url(forResource: "AppIcon", withExtension: "icns")!)` (dev-run Dock icon).
  - `Support/Info.plist`: add `CFBundleIconFile = AppIcon`.
  - `Scripts/bundle.sh`: copy `AppIcon.icns` into `RepoDeck.app/Contents/Resources/`.
- Verify: iconset builds, bundle contains the icns, `plutil -lint` passes, app launches. Visual acceptance is the owner's.

## Feature 2 — Theme model + Feature 3 — Settings window

- **`ColorHex`** (RepoDeckKit, pure, unit-tested): `encode(r,g,b,a) -> "#RRGGBBAA"` and `decode(String) -> (r,g,b,a)?`; tolerant of `#RGB`/`#RRGGBB`/`#RRGGBBAA`, case-insensitive, returns nil on malformed input. Tests cover round-trip, each length form, and malformed input.
- **`ThemeSettings`** (RepoDeck app, `@MainActor @Observable final class`), persisted to `UserDefaults.standard`:
  - `appearance: Appearance` (`.system|.light|.dark`, key `"theme.appearance"`) → computed `colorScheme: ColorScheme?` (nil for system).
  - `accentHex: String` (key `"theme.accentHex"`, default a violet like `#7A5CFFFF`) → computed `accent: Color` via `ColorHex`.
  - `uiFontName: String?` (key `"theme.uiFont"`, nil = system).
  - `monoFontName: String?` (key `"theme.monoFont"`, nil = system monospaced).
  - `baseFontSize: Double` (key `"theme.fontSize"`, default 13, range 10–20).
  - `resetToDefaults()`.
- **`Theme`** (resolved, injected via `EnvironmentKey` `\.theme`): built from `ThemeSettings`; exposes font helpers that honor family + scale relative to `baseFontSize/13`: `body`, `title`, `caption`, `caption2`, and `mono(_ style:)`, plus `accent`. Font family nil → system (`.system(...)`); non-nil → `.custom(name, size:)`.
- **Wiring in `RepoDeckApp`:** own `@State private var theme = ThemeSettings()`; inject `.environment(theme)` and `.environment(\.theme, Theme(settings: theme))` into BOTH the `WindowGroup` and a new `Settings { SettingsView() }` scene. Apply `.preferredColorScheme(theme.appearance.colorScheme)` and `.tint(theme.accent)` on the window content. The `Settings` scene provides the ⌘, menu item automatically.
- **`SettingsView`:** `Form`/`TabView` sections — Appearance (`Picker` System/Light/Dark), Accent (`ColorPicker`), Fonts (UI-family `Picker` over `NSFontManager.shared.availableFontFamilies` with a "System" default; monospace-family `Picker` over families filtered to those offering a fixed-pitch member, "System Monospaced" default), Font Size (`Slider` 10–20 with a live sample line), and a "Reset to Defaults" button.

## Feature 4 — Apply theme across views

Leaf views read `@Environment(\.theme)` and replace explicit `.font(...)` calls with theme font helpers, sized off the base: `RepoRowView`, `ChangesListView`, `FileChangeRow`, `StatusLetterBadge` (mono, scaled), `CommitBoxView`, `SyncControlsView`, `HistoryListView`, `CommitRow`. Hashes/paths use `theme.mono(...)`. Semantic status colors (M orange, A green, …) and ref-capsule tints are unchanged — accent drives buttons/selection only. No functional/behavior changes, purely presentation.

## Feature 5 — Draggable Changes⇕History splitter

- Replace the fixed `Divider()` between `ChangesListView` and `HistoryListView` in `RepoDetailView` with a `GeometryReader`-based vertical split: Changes gets `fraction × availableHeight`, History the remainder, with a draggable handle between them.
- Handle: a thin (~8pt) bar; `.onHover` pushes/pops `NSCursor.resizeUpDown`; a `DragGesture` updates the fraction, clamped to 0.15–0.85.
- Persist the fraction in `UserDefaults` key `"detail.changesFraction"` (single global value, default 0.5). Works regardless of Changes content (empty "No Changes" state still resizable).
- The commit box + sync controls above stay fixed-height; only Changes/History share the flexible region.

## Feature 6 — Searchable history

- **Model** (RepoDeckKit): `enum HistorySearchField { case message, author, path, content }`; `struct HistorySearchQuery: Sendable, Equatable { var text: String; var field: HistorySearchField }`.
- **`GitClient.searchLog(query:in:limit:)`** builds `git log -n <limit> --pretty=format:<same format>` plus, per field (case-insensitive where applicable):
  - `.message` → `--grep=<text> -i`
  - `.author` → `--author=<text> -i`
  - `.path` → append `-- <text>` pathspec
  - `.content` → `-G<text>` (regex/substring diff pickaxe)
  - Reuses `LogParser`; same exit-128 "does not have any commits" → `[]` handling. Integration tests (kit): temp repo with commits varying message/author/touched-file/added-content; assert each field returns exactly the expected commits and empty text returns the full log.
- **`RepoViewModel`:** add `historyQuery: String = ""`, `historyField: HistorySearchField = .message`. `refreshLog()` calls `searchLog` when `historyQuery` is non-empty, else `log`. Typing is debounced ~300ms (cancel-and-reschedule Task) before querying; field change re-runs immediately.
- **`HistoryListView`:** header gains a search `TextField` + a scope `Picker`/menu (Message/Author/File/Content) + result count; "No matching commits" empty state when a query yields nothing.

## Execution & verification

Serial on `main` (shared files: `RepoDeckApp.swift`, `RepoDetailView.swift`, `HistoryListView.swift`, `Package.swift`). Fresh implementer + independent reviewer per task; final whole-branch review; rebuild bundle via `Scripts/bundle.sh`; push. Ordered tasks:
1. App icon (script + iconset + icns + Package/Info.plist/bundle/AppDelegate wiring).
2. `ColorHex` (kit + tests) + `ThemeSettings` + `Theme` + `Settings` scene + `SettingsView` + app wiring (preferredColorScheme/tint).
3. Apply theme fonts/accent across leaf views.
4. Draggable Changes⇕History splitter (persisted fraction).
5. Searchable history (`HistorySearchQuery` + `GitClient.searchLog` + kit tests + `RepoViewModel` + `HistoryListView` UI).
6. Rebuild bundle + end-to-end verification + final review + push.

New automated tests: `ColorHexTests`, `GitClient.searchLog` integration tests (both in `RepoDeckKitTests`). UI (icon look, theming, splitter) is verified by build + launch + owner click-through.
